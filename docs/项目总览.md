### 三层沉浸信息（已实现）

```
乾明元年 · 春（弱冠，20岁）         ← 时间+年龄感 [ui.js getTimeDisplay]
圣上勤政，百官各怀心思。             ← 世界视角 [ui.js getSituation]
「寒窗苦读十数载，今日方踏入...」    ← 第一人称内心 [ui.js getMonologue]
```

### 核心循环（v1.3 已实现）

```
[开局] 选择性别(男/女) + 出身(4种) + 人生赛道(4条)
        ↓
[每回合] 乾明X年 · 季节 · 年龄 → 内心独白 → 选择行动(3次行动点)
        ↓
[事件] 剧情节点触发（flag 因果链）→ 两选项，决策改变人生轨迹
        ↓
[危机] 赛道专属危机事件（6-14回合间随机触发）
        ↓
[结算] 资源自然增减 → 自动存档 → 推进回合
        ↓
[结局] 多结局（由历史选择积累决定），含历史注脚+结局称谓
```

## Scope Decision（范围决策）

### 必做（MVP 核心）
- 角色创建系统（身份 × 性别选择）
- 发展路线选择（4 条路线）
- 回合制文字事件系统（每条路线最少 15 个事件节点）
- 资源数值系统（5 种资源的增减逻辑）
- 胜负判定（基础终局条件）

### 可删（迭代期再加）
- 地图系统（文字描述替代视觉地图）
- 人才招募详细系统（MVP 简化为"招募/拒绝"二选一）
- 外交系统（MVP 以事件替代独立外交界面）
- 多势力 AI 对手（MVP 用随机事件模拟压力）

### 明确不做（V1 范围外）
- 实时战斗动画
- 联机/多人模式
- 语音/配音
- 独立美术资源（非必要）

## Reasoning（第一性原则推导）

**为什么这个方向值得继续：**
1. 文字 SLG 对单人独立开发最友好——不需要美术，内容是核心竞争力，可逐步叠加。
2. 古代权谋题材在中文内容市场需求稳定，受众明确。
3. 开局身份选择 + 发展路线形成自然的重玩驱动，降低需要大量内容的压力。
4. 最小 MVP（纯文字 + 数值）可在 2-4 周内验证核心体验是否成立。

**主要风险：**
- 内容生产是瓶颈：文字量大，每条路线需要充足事件节点，否则游戏感薄。
- 缺乏"钩子"：纯文字游戏起手缺乏视觉吸引，需要文案质量过硬。

## 待讨论：技术栈选型

这是当前唯一的关键分叉点，影响所有后续开发决策。

**选项 A：Web（HTML/CSS/JS）**
- 优点：无需安装、可直接分享链接、便于快速原型
- 适合用 Ink（Inkle）或纯 JS 实现事件树

**选项 B：Python CLI / Pygame**
- 优点：开发速度快（适合会 Python 的开发者）、逻辑清晰
- 缺点：分发门槛高（需安装 Python 环境）

**选项 C：Unity（文字界面）**
- 优点：跨平台发布能力强、便于后期加入视觉元素
- 缺点：前期搭建成本高、对纯文字游戏较重

**推荐**：Web 方案（选项 A）——最低门槛、最快验证、最容易分享给玩家试玩。
