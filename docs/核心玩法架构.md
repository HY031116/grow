# 核心玩法架构设计 · 权谋录

> 设计修订版 v0.2 — 古代中国人生模拟器

---

## 设计目标

> **让玩家穿越到古代，拥有一段真实的第二人生。**
> 你可以是朝堂上翻云覆雨的权臣，也可以是招兵买马的乱世枭雄；
> 可以富可敌国操控天下财富，也可以仗剑走天涯行侠仗义。
> 没有唯一正确的人生——只有你亲手书写的命运。

---

## 六条人生赛道（核心自由度）

玩家没有强制路线，但有六条**标志性人生方向**作为参照——可以纯走一条，也可以在不同阶段转换。

| 赛道 | 核心体验 | 代表结局 |
|---|---|---|
| **忠臣路** | 在朝为官、辅佐明君、与奸臣周旋 | "千古名臣"，载入史册 |
| **奸臣路** | 弄权朝堂、排除异己、架空皇帝 | "权倾天下"，生前极盛 |
| **诸侯路** | 割据一方、招兵买马、逐鹿中原 | "开国之君"或"败于一役" |
| **造反路** | 揭竿而起、振臂一呼、改朝换代 | "开国皇帝"或"乱贼伏诛" |
| **富商路** | 垄断贸易、操控市场、富可敌国 | "天下首富"，一言九鼎 |
| **侠客路** | 闯荡江湖、行侠仗义、恩怨情仇 | "江湖传说"，名动天下 |

> **关键设计原则**：每条赛道都有对应的资源体系、行动集合和专属剧情。玩家在中途可以"跨赛道"，但需要付出代价（放弃已积累的资源或声誉）。

## 双层系统设计

### 层 1：经营模拟层（可见数值）

**资源体系（按赛道分组）：**

| 资源 | 主要赛道 | 说明 |
|---|---|---|
| 钱粮 | 全赛道 | 基础生存资源 |
| 兵力 | 诸侯/造反 | 军事基础 |
| 官位 | 忠臣/奸臣 | 朝堂地位（0-9品 → 宰相） |
| 圣眷 | 忠臣 | 皇帝信任度（影响仕途） |
| 权柄 | 奸臣 | 实际控制力（vs 圣眷对立） |
| 民心 | 造反/诸侯 | 百姓支持度（影响招兵和起事） |
| 财富 | 富商 | 独立财富积累（区别于钱粮） |
| 商路 | 富商 | 垄断的贸易线数量 |
| 武艺 | 侠客 | 个人战力 |
| 江湖名望 | 侠客 | 在江湖圈的影响力 |
| 恩义 | 侠客/全赛道 | 欠你人情的人数和质量 |
| 地盘 | 诸侯/造反 | 实际控制地域 |

> MVP 阶段简化：每条赛道只激活 **3-4 个核心资源**，其余共享"钱粮"作为通用资源。

**行动系统：**

每回合 3 个行动点，可选行动按**当前赛道解锁**：

```
【通用行动】（全赛道可用）
├── 打探消息（+情报，-钱粮）
├── 广结人缘（+人脉，耗时间）
└── 休养生息（+钱粮，+体力，恢复状态）

【官场行动】（忠臣/奸臣解锁）
├── 上奏折（触发朝堂剧情）
├── 经营党羽（+权柄，-圣眷）
├── 揭发政敌（触发弹劾事件）
├── 效忠表态（+圣眷，-权柄）
└── 科举取士（招募文官型人才）

【军事行动】（诸侯/造反解锁）
├── 征兵募兵（+兵力，-钱粮）
├── 操练军队（+战斗力，-时间）
├── 攻打某地（触发战事剧情）
├── 招降纳叛（触发投降事件）
└── 屯田备战（+钱粮上限）

【商贸行动】（富商解锁）
├── 开设商铺（+钱粮/回合，-钱粮一次性）
├── 打通商路（+商路，触发商路事件）
├── 囤积居奇（+财富，-民心）
├── 贿赂权贵（-财富，+官场关系）
└── 放贷收债（触发债务纠纷剧情）

【江湖行动】（侠客解锁）
├── 行走江湖（触发随机江湖事件）
├── 切磋武艺（+武艺）
├── 接镖护送（+钱粮，触发护镖事件）
├── 替人伸冤（+江湖名望，触发恩怨剧情）
└── 组建帮派（+组织，触发帮派剧情）
```

### 层 2：剧情叙事层（深度分支）

每个行动或被动事件触发一个**剧情节点**，包含：
- 场景描述（沉浸式第一人称 / 第三人称可选）
- 2-4 个选项，每个选项有明确后果预告
- 选择影响 Flag 状态（记录"你是什么样的人"）

**Flag 分类：**

```js
flags = {
  // 身份类
  "is_official": false,         // 是否入仕
  "is_rebel_leader": false,     // 是否宣布起义
  "is_merchant_guild_head": false, // 是否成为行会会首

  // 声誉类（连续值）
  "rep_loyal": 0,               // 忠义声誉 (0-10)
  "rep_ruthless": 0,            // 狠辣声誉 (0-10)
  "rep_benevolent": 0,          // 仁德声誉 (0-10)
  "rep_cunning": 0,             // 谋略声誉 (0-10)

  // 关系类
  "emperor_trust": 50,          // 皇帝信任度（仕途赛道核心）
  "enemy_list": [],             // 结怨势力列表
  "ally_list": [],              // 盟友势力列表

  // 剧情里程碑
  "survived_purge": false,      // 是否熬过了一次政治清洗
  "avenged_family": false,      // 是否替家人复仇
  "founded_kingdom": false,     // 是否建立了独立政权
}
```

## 核心循环（完整版）

```
【开局】
└─ 选择性别（影响部分事件文字）
└─ 选择出身（6种，决定初始资源和开放行动）
└─ 选择初始目标（自由选择赛道方向，或"随缘"）
    │
    ▼
【每回合：天下形势简报】
└─ 当前局势一句话描述（皇帝在位/乱世烽烟/盛世太平等）
└─ 资源面板更新
    │
    ▼
【行动阶段】（消耗行动点，自由选择）
└─ 每个行动解锁对应剧情节点
└─ 剧情中做出选择 → 更新 Flag
    │
    ▼
【被动事件阶段】
└─ 0-2 个世界事件（旱灾、政变、行商来访、江湖仇家上门等）
└─ 玩家响应（这也可能触发赛道转换机会）
    │
    ▼
【回合结算】
└─ 资源自然增减（基于已建设的产能和当前赛道）
└─ Flag 检查是否触发里程碑事件
└─ 胜负检查 → 进入下一回合 或 触发结局
```

## 赛道转换机制（人生转折）

这是"第二人生"感的关键设计——你不是被锁死在一条路上的。

| 转换场景 | 触发条件示例 | 后果 |
|---|---|---|
| 官员 → 造反 | 圣眷归零 + 兵力 > 20 | 失去所有官位，获得"乱贼"标签，开放造反行动 |
| 商人 → 诸侯 | 财富 > 80 + 民心 > 50 | 用财富购买军队，开始割据 |
| 侠客 → 忠臣 | 江湖名望 > 60 + 皇帝发出征召 | 接受朝廷招安，保留武艺但获得官职 |
| 造反 → 侠客 | 起义失败，只身逃脱 | 流落江湖，保留武艺，开放侠客行动 |
| 忠臣 → 奸臣 | 权柄 > 圣眷 × 2 + 政敌全部清除 | 可以选择"逼宫"剧情线 |

> **设计原则**：转换必须有代价，不能无成本切换。但转换本身不是惩罚，人生转折是游戏最戏剧性的时刻。

## 六大结局方向（对应赛道）

| 方向 | 代表结局 | 触发条件（简述） |
|---|---|---|
| 忠臣 | 千古名臣 | 高圣眷 + 高忠义声誉 + 辅佐至盛世 |
| 奸臣 | 独揽朝纲 | 高权柄 + 皇帝被架空 + 无人可制衡 |
| 奸臣 | 奸臣落幕 | 权柄极高但民心崩溃，革命来临 |
| 诸侯 | 割据称王 | 高地盘 + 高兵力 + 宣布独立 |
| 造反 | 开国皇帝 | 推翻朝廷 + 高民心 + 建立新朝 |
| 造反 | 乱贼伏诛 | 起义失败，兵力归零 |
| 富商 | 富可敌国 | 超高财富 + 垄断多条商路 + 皇帝欠债 |
| 侠客 | 江湖传说 | 高武艺 + 高江湖名望 + 行遍天下 |
| 通用 | 平淡终老 | 低调生存至游戏末期，不追求极端结局 |

## 复杂度控制（防止范围膨胀）

这套设计最大风险是**内容爆炸**。控制策略：

1. MVP 只实现 **2 条赛道完整**（忠臣/奸臣路 + 造反路），其余赛道共享通用行动框架。
2. **剧情节点分层**：主线节点 10 个（精心分支）+ 通用节点 20 个（轻度分支）+ 随机事件池 15 个。
3. **Flag 在 MVP 阶段限制 12 个**，防止状态空间爆炸。
4. **赛道转换在 MVP 只实现 2 条转换路径**（官员→造反，侠客→忠臣）。
5. 每条赛道 MVP 激活 **3 个核心资源**，其余资源在后续版本扩展。

## 里程碑定义

| 里程碑 | 目标 | 关键产出 |
|---|---|---|
| **M0** ✅ | 文档基础建立 | 项目总览 + 架构文档 |
| **M1 — 框架原型** | 行动框架 + 资源面板可在浏览器运行 | 无内容，只有结构骨架 |
| **M2 — 第一条赛道** | 官场赛道（忠臣/奸臣）完整可玩 | 10 主线节点 + Flag 系统 |
| **M3 — 第二条赛道** | 造反赛道完整 + 赛道转换机制 | 第二赛道内容 + 转换事件 |
| **M4 — MVP 完整** | 全部 6 条赛道基础框架 + 多结局 | 完整 MVP 可内测 |
